# ScholarAI Platform Features - Complete Documentation

## Architecture Overview

ScholarAI follows a **Meta Repository Architecture** with Git submodules for modular, independent service development:

### Core Components
- **Meta Repository**: Orchestration and Docker management
- **Frontend**: Next.js 14+ TypeScript application  
- **Microservices**: Spring Boot 3.5.0 Java 21 services
- **AI Services**: FastAPI Python services (extractor, paper-search)
- **Infrastructure**: PostgreSQL, Redis, RabbitMQ

---

## Technical Architecture Summary

### Backend Microservices (Java/Spring Boot)
1. **User Service** (Port 8081) - Authentication & profiles
2. **Project Service** (Port 8083) - Research project management  
3. **Notification Service** (Port 8082) - Email & alerts
4. **API Gateway** (Port 8989) - Request routing & load balancing
5. **Service Registry** (Port 8761) - Service discovery

### AI Services (Python/FastAPI)
1. **Extractor Service** (Port 8084) - PDF content extraction
2. **Paper-Search Service** (Port 8085) - Academic search & collection

### Frontend (TypeScript/Next.js)
- **Main Application** (Port 3000) - User interface
- **Server-Side Rendering** with API integration
- **Real-time updates** via WebSocket connections

### Infrastructure Services
- **PostgreSQL** - Primary data storage (per-service databases)
- **Redis** - Caching and session storage  
- **RabbitMQ** - Message queuing system
- **Backblaze B2** - Cloud file storage
